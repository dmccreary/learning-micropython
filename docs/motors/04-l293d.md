## What Is the L293D?

The **L293D** is a dual H-bridge motor driver chip. "Dual" means it can control
**two DC motors** independently — or one stepper motor.

The "D" at the end means it has built-in **flyback diodes**. These protect the
chip from the voltage spikes that motors create when they switch off.

Key specs:

- Motor voltage: **4.5 V to 36 V** (the Vcc2 pin)
- Logic voltage: **5 V** (the Vcc1 pin)
- Max current per channel: **600 mA continuous**, 1.2 A peak
- Cost: about **$0.50–$1.00** per chip

## The L293D Pinout

The L293D has 16 pins arranged as a Dual In-line Package (DIP):

| Pin | Name | Function |
|-----|------|----------|
| 1 | Enable 1,2 | HIGH to enable Motor A |
| 2 | Input 1 | Direction control for Motor A |
| 3 | Output 1 | Motor A terminal 1 |
| 6 | Output 2 | Motor A terminal 2 |
| 7 | Input 2 | Direction control for Motor A |
| 8 | Vcc2 | Motor power supply (4.5–36 V) |
| 9 | Enable 3,4 | HIGH to enable Motor B |
| 10 | Input 3 | Direction control for Motor B |
| 11 | Output 3 | Motor B terminal 1 |
| 14 | Output 4 | Motor B terminal 2 |
| 15 | Input 4 | Direction control for Motor B |
| 16 | Vcc1 | Logic power supply (5 V) |
| 4, 5, 12, 13 | GND | Ground (also act as heat sinks) |

!!! mascot-thinking "Key Idea"
    ![Monty thinking](../img/mascot/thinking.png){ class="mascot-admonition-img" }
    The L293D needs **two separate power supplies**: Vcc1 (5 V) powers the logic
    inside the chip, and Vcc2 powers the motors. This separation protects your
    Pico from noisy motor currents.

## Wiring the L293D to the Pico

Connect the L293D to one motor and the Pico like this:

| L293D Pin | Connect to | Signal |
|-----------|-----------|--------|
| Vcc1 (pin 16) | Pico VBUS (pin 40) | 5 V logic power |
| Vcc2 (pin 8) | Battery pack positive | Motor power |
| GND (pins 4, 5, 12, 13) | GND | Ground |
| Enable 1,2 (pin 1) | Pico GP20 | Enable Motor A |
| Input 1 (pin 2) | Pico GP18 | Motor A direction |
| Input 2 (pin 7) | Pico GP19 | Motor A direction |
| Output 1 (pin 3) | Motor wire 1 | Motor terminal |
| Output 2 (pin 6) | Motor wire 2 | Motor terminal |

!!! mascot-warning "Watch Out!"
    ![Monty warning](../img/mascot/warning.png){ class="mascot-admonition-img" }
    The Pico's GPIO pins run at **3.3 V**, but Vcc1 needs **5 V**. Connect Vcc1
    to the Pico's **VBUS** pin (5 V from USB), not to the 3V3 pin. The logic
    inputs (IN1, IN2) are still driven by 3.3 V GPIO pins — this works fine.

## Sample Program: Forward, Backward, Stop

```python
from machine import Pin, PWM
import time

# Enable pin — set HIGH to allow the motor to run
enable = Pin(20, Pin.OUT)
enable.value(1)        # enable the motor driver

# Direction pins for Motor A
in1 = Pin(18, Pin.OUT) # Input 1
in2 = Pin(19, Pin.OUT) # Input 2

def motor_forward():
    """Spin the motor forward."""
    in1.value(1)  # IN1 HIGH, IN2 LOW = forward
    in2.value(0)

def motor_backward():
    """Spin the motor backward."""
    in1.value(0)  # IN1 LOW, IN2 HIGH = backward
    in2.value(1)

def motor_stop():
    """Stop the motor (brake)."""
    in1.value(0)  # Both LOW = stop
    in2.value(0)

# Run a sequence: forward → stop → backward → stop
motor_forward()
time.sleep(2)

motor_stop()
time.sleep(0.5)

motor_backward()
time.sleep(2)

motor_stop()
```

## What Each Line Does

| Line | Purpose |
|------|---------|
| `enable.value(1)` | Turns on the H-bridge so current can flow to the motor |
| `in1.value(1), in2.value(0)` | Forward: current flows through the motor in one direction |
| `in1.value(0), in2.value(1)` | Backward: current flows through the motor the other direction |
| `in1.value(0), in2.value(0)` | Stop: both inputs LOW cuts power to the motor |

## Controlling Speed with PWM

Connect the **Enable** pin to a PWM-capable GPIO pin to control motor speed:

```python
from machine import Pin, PWM
import time

# Use PWM on the enable pin for speed control
enable_pwm = PWM(Pin(20))
enable_pwm.freq(1000)  # 1 kHz PWM

in1 = Pin(18, Pin.OUT)
in2 = Pin(19, Pin.OUT)

# Go forward at 50% speed
in1.value(1)
in2.value(0)
enable_pwm.duty_u16(32768)  # 32768 = 50% of 65535
time.sleep(2)

# Ramp up to full speed over 1 second
for speed in range(32768, 65536, 655):
    enable_pwm.duty_u16(speed)
    time.sleep(0.01)

# Stop
enable_pwm.duty_u16(0)
```

!!! mascot-tip "Monty's Tip"
    ![Monty giving a tip](../img/mascot/tip.png){ class="mascot-admonition-img" }
    The L293D loses about **1.5 V** across its output transistors. If you are using
    a 5 V battery, your motor only gets about 3.5 V. For more powerful motors,
    use a higher battery voltage or upgrade to the **DRV8833**, which has much
    lower losses.
